ASCOM Switch Driver (C#)



This is a basic ASCOM Switch driver in C#. It produces an in-process (assembly) based driver. This passes the ASCOM conformance test but the author offers no guarantee of its performance and the author offers no guarantee of its performance. You use at your own risk .



This uses the KMTronic relays – these have a FTDi-based USB serial port built in and the following simple protocol.

Communication Parameters:
8 Data, 1 Stop, No Parity
Baud rate : 9600

Commands:
FIRST channel commands:
OFF command : FF 01 00 (HEX) or 255 1 0 (DEC)
ON command : FF 01 01 (HEX) or 255 1 1 (DEC)

SECOND channel commands:
OFF command : FF 02 00 (HEX) or 255 2 0 (DEC)
ON command : FF 02 01 (HEX) or 255 2 1 (DEC)

THIRD channel commands:
OFF command : FF 03 00 (HEX) or 255 3 0 (DEC)
ON command : FF 03 01 (HEX) or 255 3 1 (DEC)

FOURTH channel commands:
OFF command : FF 04 00 (HEX) or 255 4 0 (DEC)
ON command : FF 04 01 (HEX) or 255 4 1 (DEC)



and so on



Status request

FF 09 00 (HEX), replies : aa bb cc dd, for the four relays aa=01 for on, 00 for off, etc.



https://info.kmtronic.com/kmtronic-usb-relay-test-software.html



NOTE: there is no way of telling the number of relays on a board through the software. It can cope with a relay count less or equal to the physical number of relays, but it will crash if you set it to a higher than actual number in the ASCOM properties.
